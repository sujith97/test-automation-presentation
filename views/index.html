What is test automation?

Test automation is the use of software (under certain preconditions) to execute tests and then determine whether the actual outcomes and the predicted outcomes are the same.

What is a test automation framework?

A test automation framework is an execution environment for automated tests. It is a sandbox in which the tests will be automated.

Capco's take on test automation framework

  • User stories are first gathererd. (Eg: From Rally)
  • They are then converted into Gherkin scripts in an Excel spreadsheet.
  • This spreadsheet is fed into the test automation framework.
  • This application is run once the glue code is implemented generating the Cucumber reports.

Features

  • Exposes an excel spreadsheet for the analyst to easily prepare Gherkin scripts. Friendly
  • Provides concise API to test backend and front end components. Usability
  • Auto generates Cucumber Java glue code right from the excel spreadsheet so that the developer/tester can start implementation quickly. Low Maintenance Quick
  • Publishes Cucumber test results. Reports Ease

Properties

  • It is application independent. Reusability
  • It is easy to expand, maintain and perpetuate Robust
